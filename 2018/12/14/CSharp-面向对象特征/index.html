C#-面向对象的特征

常用的访问修饰符

名称 说明
public 公共的，不限制对该类的访问
protected 受保护的，只能从其所在的类或子类进行访问
private 私有的，只有在.net中的的应用程序或库中才能访问
internal 内部的，只有其所在的类才能访问
abstract 抽象类，不允许建立类的实例，只允许继承类
sealed 密封类，不允许类被继承

继承

继承是面向对象程序设计中最重要的概念之一。继承允许我们根据一个类来定义另一个类，这使得创建和维护应用程序变得更容易。同时也有利于重用代码和节省开发时间。
当创建一个类时，程序员不需要完全重新编写新的数据成员和成员函数，只需要设计一个新的类，继承了已有的类的成员即可。这个已有的类被称为的基类，这个新的类被称为派生类。
当派生类从基类类继承时，派生类会获得基类的所有方法，属性，字段以及事件。
值得注意的是，c#只允许我们继承一个类，要想更多的继承，我们可以继承接口。

多态性

动态性是指类为名称相同的方法提供不同的实现方式。利用多态性，可以调用类中的某个方法而无需考虑该方法的实现过程。
类的多态性是通过派生类中的重写或隐藏基类的方法或函数成员来实现的。
值得注意的是，父类中的虚方法必须用override来重写，非虚方法使用new关键字来隐藏。
被隐藏的方法，如果实例化的是子类，则调用子类的方法，否则，调用父类的方法

抽象

抽象类

  1. 抽象类不能被实例化(new)

  2. 抽象类中可以包含抽象成员，但非抽象类不可以

  3. 抽象类不能被密封

  4. 抽象类使用abstract修饰符，并且只能作为基类使用。

抽象方法的声明

抽象方法是隐式的虚方法(virtual)，抽象方法只能在抽象类中声明。
抽象方法不能使用private static virtual 修饰符。(抽象方法默认是一个virtual方法)
抽象方法不能提供具体的实现，没有方法体。若要实现抽象方法，需要在其子类中进行重写（override）抽象方法，子类只有在实现所有抽象类的抽象方法后才能被实例化。

封装

在c#中不能被继承的类称为密封类(sealed),密封类用来限制扩展性，如果密封了某个类，则该成员不能被重写。
sealed修饰符可以用于类，实例方法，属性，事件，索引器上，但是不能用于静态成员。
sealed修饰符必须始终与override结合使用。将密封类作为基类是错误的。
在一个类中的密封成员可以用new修饰符隐藏。

接口

上面的继承中我们提到，当我们想继承多个时，我们就需要接口了。
接口可以使用interface 关键字来声明。接口名称一般以大写’I’开头
可以使用public private new internal 修饰符声明接口，但是接口的成员必须使用public来声明。
接口可以包含方法，属性，事件，索引器，四种类型，但是不能包含字段，也不能为这些成员赋值，只能定义成员。

接口的特点

  • 不能直接实例化接口。
  • 接口可以包含方法，属性，事件，索引器。
  • 接口不包含方法的实现。
  • 要继承接口，就必须实现接口的所有成员
  • 类，结构以及接口都可以继承多个接口
  • 接口成员的定义与类成员类似但是需要注意
    • 接口成员不能包含代码体
    • 接口成员只能使用public来定义
    • 接口成员不能包含类型定义(如 int string)

接口示例

接口与抽象的异同

  • 两者的派生类只能继承一个基类，即只能继承一个抽象类，但是可以继承多个接口。
  • 抽象类中可以定义成员的实现，但接口不可以。
  • 抽象类中可以包含字段，构造函数，析构函数，静态成员与常量等，接口不可以。
  • 抽象中的成员可以是私有的(只要不是抽象的)，受保护的，内部可，受保护内部成员。接口必须是公共的。
